# Лабораторна робота №24: Strategy + Observer

## Опис

Реалізація патернів проектування **Strategy** та **Observer** для системи обробки числових даних.

## Структура проєкту

### Патерн Strategy

- **INumericOperationStrategy** - інтерфейс стратегії для числових операцій
- **SquareOperationStrategy** - обчислення квадрата числа
- **CubeOperationStrategy** - обчислення куба числа
- **SquareRootOperationStrategy** - обчислення квадратного кореня
- **NumericProcessor** - клас для обробки чисел з динамічною зміною стратегій

### Патерн Observer

- **ResultPublisher** - клас-видавець з подією `ResultCalculated`
- **ConsoleLoggerObserver** - виводить результати в консоль
- **HistoryLoggerObserver** - зберігає історію обчислень
- **ThresholdNotifierObserver** - сповіщає про перевищення порогового значення

## Запуск

```bash
dotnet run
```

## Тестування

Для запуску юніт-тестів:

```bash
dotnet test
```

Тести покривають:
- Всі стратегії (квадрат, куб, квадратний корінь)
- Клас NumericProcessor
- Всі спостерігачі
- Інтеграційні тести взаємодії компонентів

## Особливості реалізації

- **Динамічна зміна стратегій** - можна змінювати алгоритм обробки під час виконання
- **Події C#** - Observer реалізований через події мови C#
- **Множинні спостерігачі** - можна підписати кілька спостерігачів на одну подію
- **Обробка помилок** - перевірка вхідних даних для квадратного кореня
