# Лекція 23. Юніт-тестування

## Домашнє завдання

Метою завдання є вивчення принципів юніт-тестування в середовищі C# та закріплення навичок створення автоматизованих тестів для перевірки коректності роботи програмного коду.

## Структура матеріалів

ESSAY_UnitTesting.md — теоретичний огляд теми юніт-тестування.
BankAccount.cs — реалізація класу BankAccount.
BankAccountTests.cs — набір юніт-тестів (16 тестів).
NOTES_AboutMocking.md — пояснення mock-об'єктів.

## Перелік реалізованих тестів

### Метод Deposit()
- збільшення балансу після коректного поповнення
- поповнення мінімальною допустимою сумою
- обробку нульового значення
- обробку від'ємного значення

### Метод Withdraw()
- зменшення балансу при коректному знятті
- зняття повної суми коштів
- спробу зняття суми, що перевищує баланс
- обробку нульового значення
- обробку від'ємного значення

### Метод TransferTo()
- успішний переказ між рахунками
- обробку нульової суми
- ситуацію недостатності коштів
- переказ усієї суми балансу
- передачу null як об'єкта призначення

### Конструктор класу
- створення об'єкта з коректними параметрами
- передачу порожнього імені
- передачу від'ємного початкового балансу
- створення рахунку з нульовим балансом

dotnet test
